## Savol 1: Fetch API da HTTP error handling qanday qilinadi? [Junior+]

**Javob:**

```javascript
// Fetch faqat TARMOQ xatosida reject bo'ladi
// 404/500 reject BO'LMAYDI!

// Noto'g'ri:
const data = await fetch("/api/users").then(r => r.json());
// 404 bo'lsa ham shu yerga keladi

// To'g'ri:
const response = await fetch("/api/users");
if (!response.ok) {
  throw new Error(`HTTP ${response.status}: ${response.statusText}`);
}
const data = await response.json();
```

`response.ok` — status 200-299 orasidami. TypeError — tarmoq xatosi (server yoqilmagan, CORS). AbortError — so'rov bekor qilingan.

## Savol 2: AbortController nima? Qanday ishlatiladi? [Middle]

**Javob:**

```javascript
const controller = new AbortController();

// Timeout
setTimeout(() => controller.abort(), 5000);

try {
  const response = await fetch("/api/data", {
    signal: controller.signal
  });
  const data = await response.json();
} catch (error) {
  if (error.name === "AbortError") {
    console.log("Bekor qilindi");
  }
}
```

```javascript
// Timeout helper
function fetchWithTimeout(url, options = {}, timeout = 5000) {
  const controller = new AbortController();
  const id = setTimeout(() => controller.abort(), timeout);
  return fetch(url, { ...options, signal: controller.signal })
    .finally(() => clearTimeout(id));
}
```

**Use cases:** timeout, user cancel, search debounce (eski request bekor qilish), SPA navigatsiya (oldingi sahifa fetch'ini bekor qilish).

## Savol 3: `localStorage` vs `sessionStorage` vs `cookies` farqi [Junior+]

**Javob:**

|---|---|---|---|
| Hajm | ~5-10MB | ~5-10MB | ~4KB |


```javascript
  console.log(`${e.key}: ${e.oldValue} → ${e.newValue}`);
```

## Savol 4: IntersectionObserver nima? Lazy loading qanday qilinadi? [Middle]

**Javob:**

Element viewport ga ko'ringanini kuzatadi — scroll handler kerak emas.

```javascript
function lazyLoadImages() {
  const images = document.querySelectorAll("img[data-src]");

  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        const img = entry.target;
        img.src = img.dataset.src;
        img.removeAttribute("data-src");
        observer.unobserve(img); // kuzatishni to'xtatish
      }
    });
  }, {
    rootMargin: "200px" // 200px oldindan yuklash
  });

  images.forEach(img => observer.observe(img));
}
```

**Options:**
- `root` — null = viewport, yoki boshqa element
- `rootMargin` — preload margin
- `threshold` — 0..1, qancha foiz ko'ringanda trigger

## Savol 5: Web Worker nima? Qachon ishlatiladi? [Middle]

**Javob:**

JavaScript ni **alohida thread** da ishlatish — main thread (UI) bloklanmaydi.

```javascript
// main.js
const worker = new Worker("worker.js");
worker.postMessage({ data: [1, 2, 3] });
worker.onmessage = (e) => console.log("Natija:", e.data);

// worker.js
self.onmessage = (e) => {
  const result = e.data.data.reduce((s, n) => s + n, 0);
  self.postMessage(result);
};
```

**Worker da MUMKIN EMAS:** `document`, `window`, `alert`, `localStorage`.
**MUMKIN:** `fetch`, `setTimeout`, `IndexedDB`, `WebSocket`, `console`.

**Qachon:** og'ir hisoblash, katta JSON parse, rasm processing, video encoding.

## Savol 7: WebSocket vs SSE (Server-Sent Events) farqi [Middle+]

**Javob:**

| | WebSocket | SSE (EventSource) |
|---|-----------|-------------------|
| Yo'nalish | Ikki tomonlama | Faqat server → client |
| Protocol | ws:// / wss:// | HTTP |
| Reconnect | Qo'lda | Avtomatik |
| Binary data | Ha | Yo'q (text only) |
| Ishlatish | Chat, o'yinlar, collaborative | Live feed, notifications |

```javascript
// WebSocket — ikki tomonlama
const ws = new WebSocket("wss://example.com/socket");
ws.send("Salom"); // client → server
ws.onmessage = (e) => console.log(e.data); // server → client

// SSE — faqat server → client
const source = new EventSource("/api/stream");
source.onmessage = (e) => console.log(e.data);
// source.send() — YO'Q!
```

## Savol 8: MutationObserver nima uchun kerak? [Middle]

**Javob:**

DOM o'zgarishlarini kuzatish — element qo'shildi/o'chirildi, attribute o'zgardi, matn o'zgardi.

```javascript
const observer = new MutationObserver((mutations) => {
  mutations.forEach(m => {
    if (m.type === "childList") {
      m.addedNodes.forEach(n => console.log("Qo'shildi:", n));
    }
    if (m.type === "attributes") {
      console.log(`${m.attributeName} o'zgardi`);
    }
  });
});

observer.observe(document.getElementById("app"), {
  childList: true,
  attributes: true,
  subtree: true // recursive
});
```

**Use cases:** third-party script monitoring, dynamic content tracking, custom framework reactivity.

## Savol 11: Blob va File API qanday ishlaydi? [Middle]

**Javob:**

```javascript
// Blob — binary data container
const blob = new Blob(["Salom, dunyo!"], { type: "text/plain" });

input.addEventListener("change", async (e) => {

  const a = document.createElement("a");
  a.href = URL.createObjectURL(blob);
  a.download = filename;
  a.click();
  URL.revokeObjectURL(a.href);
}
```

## Savol 12: Transferable Objects nima? [Senior]

**Javob:**

`postMessage` default **Structured Clone** (nusxa) yaratadi — katta data uchun sekin. **Transferable** — egalikni O'TKAZADI, nusxa yaratmaydi.

```javascript
const buffer = new ArrayBuffer(1024 * 1024); // 1MB
console.log(buffer.byteLength); // 1048576

worker.postMessage(buffer, [buffer]); // TRANSFER
console.log(buffer.byteLength); // 0 — main thread da bo'sh!
```

Transferable bo'lishi mumkin: `ArrayBuffer`, `MessagePort`, `ImageBitmap`, `OffscreenCanvas`.

## Savol 13: `performance.now()` vs `Date.now()` farqi [Junior+]

**Javob:**

```javascript
// Date.now() — millisecond aniqlikda, epoch dan
console.log(Date.now()); // 1709913600000

// performance.now() — microsecond aniqlikda, sahifa yuklangandan
console.log(performance.now()); // 1234.567890123
```

| | `Date.now()` | `performance.now()` |
|---|---|---|
| Aniqlik | millisecond | microsecond |
| Boshlang'ich | Unix epoch | Sahifa yuklangan vaqt |
| Monotonic | Yo'q (tizim soati o'zgarsa) | Ha |

Benchmark uchun `performance.now()` ishlatilsin — aniqroq va tizim soati o'zgarishiga bog'liq emas.

## Savol 14: Cookie attributes nima? CSRF himoya qanday? [Middle+]

**Javob:**

```javascript
document.cookie = "token=abc; max-age=86400; path=/; secure; samesite=strict";
```

| Attribute | Vazifasi |
|-----------|----------|
| `max-age` | Necha soniya yashaydi |
| `expires` | Aniq sana |
| `path` | Qaysi path uchun |
| `domain` | Qaysi domain uchun |
| `secure` | Faqat HTTPS |
| `httpOnly` | JS dan o'qib bo'lmaydi (server qo'yadi) |
| `samesite` | CSRF himoya: strict, lax, none |

`samesite=strict` — cookie faqat same-site so'rovlarda yuboriladi (cross-site form submit, link click — cookie yuborilmaydi). `httpOnly` — `document.cookie` da ko'rinmaydi, faqat HTTP header'da.

## Savol 15: Coding: Fetch wrapper bilan retry logic [Middle+]

**Javob:**

```javascript
async function fetchWithRetry(url, options = {}, retries = 3) {
  for (let i = 0; i <= retries; i++) {
    try {
      const response = await fetchWithTimeout(url, options, 5000);
      if (!response.ok) {
        throw new Error(`HTTP ${response.status}`);
      }
      return await response.json();
    } catch (error) {
      if (i === retries) throw error;

      // Exponential backoff
      const delay = Math.min(1000 * 2 ** i, 10000);
      console.log(`Retry ${i + 1}/${retries} in ${delay}ms`);
      await new Promise(r => setTimeout(r, delay));
    }
  }
}

function fetchWithTimeout(url, options, timeout) {
  const controller = new AbortController();
  const id = setTimeout(() => controller.abort(), timeout);
  return fetch(url, { ...options, signal: controller.signal })
    .finally(() => clearTimeout(id));
}
```

**Kalit nuqtalar:**
- Exponential backoff — har retry kutish vaqti ikki baravar
- `fetchWithTimeout` — AbortController bilan timeout
- `response.ok` tekshirish — HTTP xatolar uchun

## Savol 17: Coding: Infinite Scroll implement qiling [Middle+]

**Javob:**

```javascript
function setupInfiniteScroll(container, loadMore) {
  const sentinel = document.createElement("div");
  sentinel.id = "sentinel";
  container.appendChild(sentinel);

  let loading = false;

  const observer = new IntersectionObserver(async (entries) => {
    if (!entries[0].isIntersecting || loading) return;

    loading = true;
    const hasMore = await loadMore();
    loading = false;

    if (!hasMore) {
      observer.unobserve(sentinel);
      sentinel.remove();
    }
  }, { rootMargin: "200px" });

  observer.observe(sentinel);

  return () => {
    observer.disconnect();
    sentinel.remove();
  };
}

// Ishlatish:
let page = 1;
setupInfiniteScroll(document.getElementById("feed"), async () => {
  const data = await fetch(`/api/posts?page=${page++}`).then(r => r.json());
  data.posts.forEach(post => {
    const el = document.createElement("article");
    el.textContent = post.title;
    document.getElementById("feed").insertBefore(el,
      document.getElementById("sentinel"));
  });
  return data.posts.length > 0;
});
```

## Savol 18: Service Worker qanday ishlaydi? Lifecycle? [Senior]

**Javob:**

```
Register → Install (cache yaratish) → Wait (eski SW bo'lsa) → Activate → Fetch events
```

```javascript
// Registratsiya
navigator.serviceWorker.register("/sw.js");

// sw.js
self.addEventListener("install", (e) => {
  e.waitUntil(
    caches.open("v1").then(c => c.addAll(["/", "/app.js", "/style.css"]))
  );
});

self.addEventListener("fetch", (e) => {
  e.respondWith(
    caches.match(e.request).then(r => r || fetch(e.request))
  );
});
```

**Cache strategiyalari:**
- **Cache First** — cache da bo'lsa qaytaradi, yo'q bo'lsa fetch
- **Network First** — avval fetch, xato bo'lsa cache
- **Stale While Revalidate** — cache dan qaytaradi, background da yangilaydi

HTTPS talab qiladi (localhost bundan mustasno).

## Savol 19: Web Crypto API nima uchun kerak? [Middle]

**Javob:**

```javascript
// Random qiymatlar (Math.random() dan xavfsizroq)
const array = new Uint32Array(10);
crypto.getRandomValues(array);

// UUID
const uuid = crypto.randomUUID();

// SHA-256 hash
async function sha256(message) {
  const data = new TextEncoder().encode(message);
  const hash = await crypto.subtle.digest("SHA-256", data);
  return Array.from(new Uint8Array(hash))
    .map(b => b.toString(16).padStart(2, "0"))
    .join("");
}

await sha256("salom"); // "abc123..."
```

`crypto.getRandomValues()` — kriptografik xavfsiz random. `Math.random()` — pseudo-random, xavfsiz emas. Token, ID, va shifrlash uchun `crypto` ishlatilsin.

## Savol 20: IndexedDB qachon localStorage o'rniga ishlatiladi? [Middle]

**Javob:**

| | localStorage | IndexedDB |
|---|---|---|
| Hajm | ~5-10MB | ~250MB+ |
| Data turi | Faqat string | Har qanday (object, blob, file) |
| API | Sinxron | Asinxron (main thread bloklamaydi) |
| Index/search | Yo'q | Ha (index, cursor, range query) |
| Transaction | Yo'q | Ha (ACID) |

**IndexedDB qachon:** katta data (offline app, file cache), murakkab query kerak (index, filter), structured data (object store), asinxron kerak (katta data sinxron o'qish UI ni bloklaydi).

**localStorage qachon:** kichik key-value (theme, settings, token), oddiy saqlash.
